PHP中的哈希表结构及HASH算法

发布于: February 29, 2012, 10:26 am 分类: PHP原理 作者: Cyrec 阅读: [504]

PHP中的哈希表是十分重要的是一个数据接口，基本上大部分的语言特性都是基于哈希表的，例如：变量的作

用域和变量的存储，类的实现以及Zend引擎内部的数据有很多都是保存在哈希表中的。PHP中的哈希表使用也

链接法来避免冲突(将映射到同一位置的值用双向链表保存)。PHP使用如下两个数据结构来实现哈希表，

HashTable结构体用于保存整个哈希表需要的基本信息，而Bucket结构体用于保存具体的数据内容，

如下(摘自TIPI)：

31} Bucket;

PHP的Hash采用的是目前最为普遍的DJBX33A (Daniel J. Bernstein, Times 33 with Addition), 这个算法被广泛

运用与多个软件项目,Apache, Perl和Berkeley DB等. 对于字符串而言这是目前所知道的最好的哈希算法，原因

在于该算法的速度非常快，而且分类非常好(冲突小,分布均匀).具体的介绍在这:PHP中的Hash算法

算法的核心思想就是:hash(i) = hash(i-1) * 33 + str[i]

使用PHP模拟实现:

1function times33($string)
2{
3 $code = 5381;
4 for ($i = 0, $len = strlen($string); $i < $len; $i++) {
5  $code = (int) (($code << 5) + $code + ord($string{$i})) & 0x7fffffff;
6                //与0x7fffffff做与运算是为了防止内存溢出
7 }
8 return $code;
9}

通过times33算法将字符串产生的hash数字与hash表的大小求余就能得到比较均匀的hash分布。在PHP中有个

这样的宏：#define HASH_INDEX(ht, key) (hash_str((key)) % (ht)->size)
